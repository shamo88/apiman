# Apiman 产品优化建议

> 生成时间: 2026-04-09  
> 分析维度: 功能优化 + 产品优化

---

## 一、项目现状概览

### 技术栈
| 层级 | 技术 |
|------|------|
| 桌面框架 | Wails v2.11.0 (Go + React) |
| 前端 | React 18.2 + TypeScript 4.6 + Vite 3.x |
| UI库 | Ant Design 6.3.4 |
| 状态管理 | Zustand (7个Store) |
| 脚本引擎 | goja (JavaScript VM) |
| 数据格式 | Postman v2.1 (collection.postman.json) |

### 核心功能现状
- ✅ 多项目管理、项目分组、文件夹组织
- ✅ HTTP 请求编辑（7种方法、5种Body类型）
- ✅ 环境变量管理（多环境切换）
- ✅ 请求用例（每个请求多个测试场景）
- ✅ JavaScript 脚本支持（Pre/Post Script）
- ✅ 请求历史记录
- ✅ Git 同步备份
- ✅ MCP 服务器
- ✅ 深色/浅色主题
- ❌ 缺少快捷键系统
- ❌ 缺少导入/导出（OpenAPI等）
- ❌ 缺少批量执行
- ❌ 缺少 Mock Server

## 二、功能层面优化建议

### 2.1 缺失的核心功能

| 功能 | 现状 | 建议 | 优先级 |
|------|------|------|--------|
| **API导入/导出** | 仅支持Postman格式 | 增加 OpenAPI/Swagger 导入、导出为 Postman/Insomnia 格式 | 🔴 高 |
| **批量执行** | 无 | 支持选择多个请求按顺序/并发执行，生成测试报告 | 🔴 高 |
| **请求分组/标签** | 仅有文件夹 | 增加标签/分组功能，跨文件夹组织API | 🟡 中 |
| **参数化数据** | 无 | 支持 CSV/JSON 数据文件驱动，批量测试 | 🟡 中 |
| **Mock Server** | 无 | 内置 Mock 功能，无需后端即可测试 | 🟡 中 |
| **证书管理** | 无 | 支持客户端证书、自定义CA配置 | 🟡 中 |
| **WebSocket/GraphQL** | 仅HTTP | 扩展支持 WebSocket、GraphQL、SSE | 🟡 中 |

### 2.2 现有功能增强

#### 用例(Cases)系统
**现状**: 每个请求支持多个测试场景

**问题**:
- 快速切换用例的 UI 需要打开 modal
- 用例间的变量传递未实现
- 缺少用例复制/导入导出

**建议**:
- 将用例切换改为 Tab 栏形式，无需打开 modal
- 增加用例复制功能
- 支持用例的导入导出

#### 脚本系统
**现状**: Pre/Post scripts 功能已有，基于 goja

**建议**:
- 增加脚本市场或模板库
- 常用脚本模板（认证、加密、参数处理等）
- 完善 `am` API 文档，增加更多示例

#### 历史记录
**现状**: HistoryModal 已存在，功能较基础

**问题**:
- 搜索条件过多，界面拥挤
- 缺少批量操作

**建议**:
- 简化搜索为下拉过滤（项目 → 请求名 → 状态码）
- 增加时间范围快捷按钮（今天/本周/本月）
- 支持批量重新执行
- 增加响应时间趋势图

## 三、产品层面优化建议

### 3.1 用户体验

#### 3.1.1 首次使用门槛

| 问题 | 建议 |
|------|------|
| 无 onboarding 引导 | 增加首次使用引导，创建示例项目 |
| Empty State 缺乏功能介绍 | 展示功能简介轮播或视频 |
| 项目无请求时无引导 | 展示示例请求结构，点击可快速创建 |

#### 3.1.2 快捷键系统

**竞品对比**: Postman 用户已习惯快捷键体系

| 快捷键 | 功能 |
|--------|------|
| `Cmd/Ctrl + Enter` | 发送请求 |
| `Cmd/Ctrl + N` | 新建请求 |
| `Cmd/Ctrl + S` | 保存 |
| `Cmd/Ctrl + Shift + N` | 新建文件夹 |
| `Cmd/Ctrl + E` | 环境切换下拉 |
| `Cmd/Ctrl + F` | 全局搜索 |
| `Cmd/Ctrl + Tab` | 切换请求标签 |

#### 3.1.3 搜索能力

**现状**: 仅支持项目内搜索

**建议**:
- 增加全局搜索（所有项目）
- 支持正则表达式
- 搜索历史记录
- 搜索范围扩展到响应内容

#### 3.1.4 响应预览增强

| 当前 | 建议 |
|------|------|
| JSON 纯文本显示 | 增加 JSON Tree View（可折叠） |
| 仅状态码 | 增加状态码说明（301 重定向等） |
| 无响应时间图表 | 增加响应时间趋势图 |

### 3.2 布局与交互

#### 3.2.1 面板布局

| 问题 | 建议 |
|------|------|
| 侧边栏宽度固定 280px | 增加拖拽调整宽度的分割线 |
| Request/Response 比例固定 | 增加拖拽分割线或快捷按钮（50:50 / 70:30 / 30:70） |
| 拖拽缺乏视觉反馈 | 添加放置位置指示线、高亮效果 |

#### 3.2.2 环境切换

**现状**: 环境切换需打开侧边栏

**建议**: 在 TitleBar 或 Footer 增加环境快速切换下拉

#### 3.2.3 HomePage 改进

| 问题 | 建议 |
|------|------|
| 无「最近项目」区域 | 增加 Recent 栏，显示最近 5 个项目 |
| 缺少快捷操作 | 右下角增加「新建项目」「导入」快捷入口 |
| 无项目时界面空旷 | 增加功能介绍引导 |

### 3.3 协作与同步

#### Git 同步
**现状**: 仅备份性质

**建议**:
- 增加 Git 分支支持
- 冲突解决 UI
- 同步状态指示器
- 多仓库关联

#### 云端同步
**竞品**: Apifox Cloud

**建议** (可选):
- 团队共享工作空间
- 云端配置同步
- 团队协作功能

### 3.4 生态与扩展

| 方向 | 现状 | 建议 |
|------|------|------|
| **插件系统** | 无 | 参考 Postman，增加插件扩展能力 |
| **MCP** | 已实现 | 增加更多 MCP tools，文档生成，AI 辅助请求生成 |
| **CLI工具** | 无 | 提供命令行工具，支持 CI/CD 集成 |
| **导入/导出** | 仅 Postman | 增加 OpenAPI/Swagger/HAR/Insomnia 格式支持 |

### 3.5 产品细节

| 功能 | 现状 | 建议 |
|------|------|------|
| 通知系统 | 无 | 请求失败、超时、脚本错误等桌面通知 |
| 多语言 | 混合 | 增加国际化支持（中/英/日/韩等） |
| 主题 | dark/light | 增加更多配色或自定义主题 |
| 日志 | 已有 | 增加 UI 日志查看器 |
| 窗口 | 单一 | 支持请求标签页独立窗口 |

### 4.3 主题系统

**实现方式**: CSS 变量 + `.theme-dark` 类名覆盖

**问题**:
- 暗色主题覆盖过于依赖 `!important`，维护困难
- 滚动条样式在暗色模式下实现分散
- 部分第三方组件暗色主题支持不完整

**建议**:
- 抽取为独立 CSS 模块
- 减少 `!important` 依赖
- 增加主题切换动画

## 五、差异化竞争点

相比 Postman/Apifox，Apiman 可以强调：

| 亮点 | 现状 | 强化方向 |
|------|------|----------|
| **MCP 原生支持** | 已实现 | 增加更多 MCP tools，文档生成，AI 辅助请求生成 |
| **请求用例(Case)** | 已有 | 强化用例管理，批量执行，用例模板市场 |
| **Wails 轻量** | 已有 | 强调启动速度、资源占用对比 Electron |
| **本地优先** | 已有 | 数据完全本地，强调隐私安全 |
| **开源可控** | 待定 | 如开源，企业用户更放心 |

## 七、附录

### 7.1 前端组件结构
```
frontend/src/components/
├── TitleBar/         # 窗口标题栏 + 设置
├── HomePage/         # 项目列表页
├── ProjectWorkspace/ # 核心工作区
├── ApiTree/          # 接口树形结构
├── RequestPanel/      # 请求编辑面板
├── ResponsePanel/     # 响应查看面板
├── ScriptHelp/        # 脚本帮助浮窗
├── MCPSettings/       # MCP 配置
├── VariableInput/     # 变量输入组件
├── AppFooter/        # 底部状态栏
└── modals/           # 13个弹窗组件
```

### 7.2 Go 后端模块
```
internal/
├── curl/      # HTTP 请求执行
├── script/    # 脚本运行时 (goja)
├── project/   # 项目管理
├── service/   # 业务逻辑编排
├── config/    # 配置管理
├── models/    # 数据结构
├── mcp/       # MCP 服务器
├── git/       # Git 同步
├── history/   # 历史记录
└── postman/  # Postman 格式兼容
```

### 7.3 状态管理 (Zustand)
| Store | 职责 |
|-------|------|
| useProjectStore | 项目列表、tabs、树、分组 |
| useWorkspaceStore | 请求tabs、当前请求、响应 |
| useEnvironmentStore | 环境列表、变量 |
| useUIStore | 模态框、拖拽、主题 |
| useScriptStore | 脚本列表 |
| useHistoryStore | 历史记录 |
